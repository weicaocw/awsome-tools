# 纳米级阅读 R 项目 · 通用 Prompt 模板

> **使用场景**：当你作为 Python / Go 等其他语言背景的程序员，想要系统性、深度地阅读一个陌生的 R 项目（含 mlr3 / torch 生态、R6 面向对象代码、data.table 等）时，将以下 prompt 框架交给 LLM，作为你的"专家导师"。

---

## 角色设定

```
你是一位耐心的 R 语言大师，同时也是 <项目作者/生态维护者> 本人。
你精通：R 语言本身（S3/S4/R6/R7 面向对象体系）、<核心依赖包列表>、
PyTorch / LibTorch 底层原理、机器学习数学基础、软件工程设计模式。

在回答所有问题时，你需要做到：
1. 对比我熟悉的语言（Python / Go）给出类比；
2. 数学公式必须完整列出，解释每个符号含义，并说明"公式讲了什么故事"；
3. 对 R 语言的"魔法语法"和"语法糖"，必须停下来单独解释，并给出至少 3 个生活化的代码示例；
4. 代码如果是"code golf"风格（挤在一起），先帮我拆分成有意义的代码模块，再逐模块解释；
5. 提到任何函数/类，必须说明：来自哪个包、输入输出类型、参数含义、底层机制；
6. 任何细枝末节都不要省略。
```

## 第一层：项目结构通览

### Prompt

```
通览这个 R 项目的目录结构，无遗漏地介绍每个目录和文件的作用。

重点回答：
1. R/ 目录下的文件加载顺序规则是什么？为什么 zzz.R 总是最后加载？
2. DESCRIPTION 和 NAMESPACE 文件分别是做什么的？逐行解读这两个文件。
   - Imports 和 Suggests 有什么本质区别？为什么某个包在 Suggests 而非 Imports？
   - importFrom(pkg, func) 的两个参数分别是什么意思？导入之后，该包其他函数还能用吗？
3. tests/ 目录的结构规范：testthat.R 和 testthat/ 子目录下的 test-*.R 分别承担什么角色？
4. 如果有 C++ 代码，它会在哪个目录？R 和 C++ 如何互相调用？
5. build.sh / Makefile 等构建脚本做了哪些事？逐行解释。
6. zzz.R 里的 .onLoad() 函数做了什么？mlr3misc::register_namespace_callback 的完整流程是什么？
7. 这个包和 <生态名> 的融合程度如何？我应该关注哪些文件来确认这一点？
```

## 第二层：核心代码纳米级阅读

### 2.1 类与对象（面向对象体系）

```
针对以下代码，帮我做纳米级解读：

<粘贴代码>

回答：
1. S3 / S4 / R6 / R7 的发展脉络是什么？各自解决了什么问题？这里用的是哪种体系？
2. 这个函数/类是"工厂函数"还是直接返回实例？有没有隐含 return？
3. 类的继承（inherit=）继承了哪个父类？父类预定义了哪些方法？这里重写了哪些？
   - 和 Python 的 super().__init__() 机制对比，R 这里的 super$initialize() 做了什么？
4. initialize / .iter / .length / set_batch_list 等方法各自的职责是什么？
5. 为什么没有 public / private / active 等关键字？（对比标准 R6 写法）
6. self$xxx <<- 和 self$xxx <- 有什么区别？闭包的"记忆"机制是如何实现的？
7. 类的继承 vs 组合的设计哲学，结合生活中 3 个例子说明。Go 语言是如何处理这个问题的？
```

### 2.2 函数与语法

```
针对以下代码片段，帮我逐行解读。遇到我作为 Python 程序员不熟悉的 R 语法，停下来单独解释：

<粘贴代码>

需要特别关注：
1. 赋值符号：<- / <<- / = / := 分别是什么，使用场景有何不同？
2. 双中括号 [[]] 和单中括号 [] 的区别。
3. 管道操作：|> 或 %>% 是什么？
4. data.table 特有语法 DT[i, j, by] 的三个槽位分别是什么意思？
   - j 槽位里直接写 data.table(...) 是什么高级用法？
   - := 赋值和 = 赋值在 data.table 里有什么区别？
5. ~ 波浪号在 dcast / melt / facet_grid 等场景中分别是什么含义？
   结合 SQL 的 GROUP BY 和 PIVOT 语法类比解释，并给出至少 3 个生活化例子。
6. %/% 整除、%% 取余、%in% 成员判断——列举 R 的常见中缀运算符。
7. ... (dot-dot-dot) 和 Python 的 *args, **kwargs 对比。
8. if(条件) 值 这种单行 if 表达式，如果条件为 FALSE 返回什么？R 里的 NULL 和 Python 的 None 对比。
9. `` 反引号括起来的符号（如 `:=`）是什么意思？什么情况下需要用反引号？
```

## 使用说明

1. **初次使用**：将"角色设定"部分发给 LLM，建立对话角色。
2. **逐层推进**：按照第一层→第六层的顺序，每次粘贴一个层级的 prompt，附上对应的代码片段。
3. **随时追问**：遇到不理解的回答，从"附录：常用追问模板"中选取合适的追问方式。
4. **替换占位符**：将 `<粘贴代码>`、`<项目作者>`、`<核心依赖包列表>` 等占位符替换为实际内容。
5. **迭代深入**：对同一段代码，可以先用"宏观解释"，再用"逐行解释"，最后用"追问细节"三轮推进。
