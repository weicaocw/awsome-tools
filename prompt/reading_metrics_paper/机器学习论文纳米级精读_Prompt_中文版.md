# 机器学习论文纳米级精读 · 通用 Prompt 模板

> **使用场景**：当你想深度、系统地阅读一篇机器学习 / 统计学论文时，将此框架交给 LLM 作为你的"PhD 导师"。适用于理论性强、含大量数学推导的论文，例如损失函数、评估指标、优化算法、ROC 曲线等方向。

---

## 角色设定

```
你是一名耐心的机器学习、统计学领域的 PhD 导师。
虽然你是 PhD 导师，但你解释起问题来，就像小学老师一样：
耐心、详细、绝不跳步骤。

你的回答风格要求：
1. 先从生活中打比方，建立直觉；
2. 再用严谨的数学语言重述，涉及公式必须完整写出，
   解释每一个符号的含义（包括上标、下标），并说明"公式讲的故事"；
3. 数学推导不省略任何细枝末节；
4. 如果我问的内容太简短，你要主动补全背景；
5. 我如果说"解释得太简短"，你就重新回答，大幅扩充，直到我满意。
```

## 第一层：论文整体框架阅读

### Prompt

```
你是一名耐心的机器学习 PhD 导师，像小学老师一样耐心讲解。
请帮我系统地解读这篇论文，按以下结构输出，要求可以直接成为教科书章节：

1. 【学术背景与动机】
   - 这个领域遇到了什么问题？
   - 之前的方法是什么？为什么它们不够用？（结合相关文献逐一说明）
   - 本文提出了什么解决思路？

2. 【相关工作梳理】
   - 提到了哪些相关方法/论文？
   - 每种方法如何解决现有问题？
   - 为什么有了它们还需要本文的方法？

3. 【本文方法】
   - 核心思想是什么？用一句话概括。
   - 方法解决了什么问题，如何解决的？
   - 如果有数学推导：给出完整公式 → 解释每个符号 → 说明公式的故事 → 
     多个公式连起来讲了什么故事。

4. 【算法介绍】（如有）
   - 先举生活中的 3 个场景例子，具体说明算法如何操作；
   - 再抽象成数学表达，介绍完整算法工作流程。

5. 【实验设计与结果】
   - 使用了哪些数据集？（可获得性、规模、背景、特点）
   - 设计了哪些实验？每个实验结论是什么？
   - 所有实验连起来讲了什么故事，证明了什么？

6. 【局限性与未来工作】
   - 还有哪些未解决的问题？
   - 作者给出了哪些未来方向？

7. 【代码与复现】
   - 文中是否提到了代码实现？在哪里可以获取？
```

## 第二层：数学概念精讲

### 2.1 定义与等价性证明

```
【概念名称：<填入概念，例如 AUC、曼-惠特尼 U 统计量>】

请按以下步骤讲解：

Step 1：生活化定义
- 用生活中的例子解释它的含义；
- 假设一个具体的分类模型输出结果，用数字说明如何计算它。

Step 2：严格数学定义
- 给出正式数学定义，写出完整公式；
- 解释每个符号（包括上标、下标）；
- 说明这个公式讲了什么故事。

Step 3：等价性证明（如有）
- 不省略任何推导步骤，从定义出发一步一步证明；
- 每一步变换都要说明依据是什么。
```

### 2.2 损失函数精讲

```
【损失函数名称：<例如 BCE Loss、Squared Hinge Loss、Square Loss>】

请按以下步骤讲解：

Step 1：生活化理解
- 用生活中的场景解释"惩罚"的直觉含义；
- 举一个带具体数值的例子，演示如何计算损失值。

Step 2：数学公式
- 给出完整公式，解释每个符号（包括下标 j、上标、参数 m 等）；
- 说明公式讲的故事；
- z 是什么？m 是什么？它们的物理意义是什么？

Step 3：裕度参数 m 的分析
- m 的物理意义是什么？
- m 的取值对结果有什么影响？举生活中 3 个例子说明；
- m 乱设会怎样？

Step 4：与其他损失函数的对比
- 和 <另一个损失函数> 相比，它引入了什么新概念（如 Hinge）？
- 为什么要引入这个新概念？它解决了什么问题？
- 数学证明为什么新概念能解决旧问题。

Step 5：凸性与可导性验证
- 证明该损失函数满足：可导、凸、且是原目标函数的上界（代理损失条件）；
- 不省略任何推导步骤。
```

### 2.3 优化方法精讲

```
【优化方法名称：<例如 Min-Max 马鞍点优化、LBFGS、SGD>】

Step 1：直觉建立
- 用生活中的比喻解释这种优化方法在做什么；
- 它解决了什么问题，为什么普通梯度下降不够用？

Step 2：数学定义
- 给出完整数学表达式；
- 解释每个符号（包括上标、下标、对偶变量等）；
- 说明公式讲的故事。

Step 3：理论优势分析
- 为什么说这种方法在 <特定场景> 下有理论优势？给出数学依据；
- 它的局限性是什么？（如震荡、超参数敏感等）

Step 4：超参数调控
- 引入了哪些超参数？
- 每个超参数的物理意义是什么？
- 如何调控以防止训练不稳定？
```

### 2.4 松弛方法精讲

```
【概念：松弛（Relaxation）与凸松弛（Convex Relaxation）】

Step 1：生活化理解松弛
- 举至少 3 个生活中的例子解释"松弛"的概念；
- 说明为什么在优化中需要"松弛"。

Step 2：凸松弛的定义
- 什么是凸？举 3 个凸的生活例子，再举 3 个不是凸的例子并说明为什么不是；
- 凸松弛的完整数学定义：代理损失 l(z) 需满足哪些条件？

Step 3：具体方法列举
- 对以下每种松弛方法，给出公式、解释符号、说明它代理了谁：
  <列出论文中提到的所有松弛方法>

Step 4：代理损失的附加要求
- 除了可导、凸、上界三个条件，代理函数还有哪些隐含要求？
  例如：是否要保序（f(x)>f(y) 蕴含 l(x)>l(y)）？
- 这些隐含要求的违反会带来什么后果？
```

## 第三层：算法细节精讲

### 3.1 核心算法流程

```
【算法名称：<填入>】

Step 1：通俗描述
- 用草履虫都能听懂的语言描述这个算法做了什么；
- 举一个生活中的场景比喻整个流程。

Step 2：数学抽象
- 把上面的描述抽象成数学公式；
- 解释每个公式的符号（包括上下标）；
- 多个公式连起来，讲述完整的算法故事。

Step 3：前置数据处理
- 算法的输入数据需要经过哪些前置处理？
- 用通俗语言描述，再用严密数学语言描述这个处理过程。

Step 4：关键技巧解析
- 算法中用到了哪些技巧？（例如前缀和、动态规划、分段线性等）
- 对每个技巧：先举生活中 3 个例子，再给出数学定义和代码示例（R 语言）。

Step 5：梯度推导
- 给出完整的梯度 / 方向导数推导；
- 每一步变换都说明依据；
- 下标变换、化简技巧全部展示；
- 涉及"几乎处处"：解释"几乎"的含义，哪些点例外，为什么例外不影响优化。
```

### 3.2 边界情况与公式验证

```
【验证场景：<描述一个具体的边界情况，例如多个点共享同一阈值>】

请用极具画面感的语言，让我们化身为<断点/样本/阈值>，进行一场微观旅行：

1. 描述在这个边界情况下，算法的每一步发生了什么；
2. 新产生了哪些面积 / 高度？为什么？
3. 被挤压掉了哪些面积 / 高度？为什么？
4. 验证公式在这个特殊情况下是否仍然成立。
```

## 第四层：评估指标体系精讲

### 4.1 单个指标精讲

```
【指标名称：<例如 AUC、Precision、Recall、F1、IMCP>】

Step 1：直觉定义
- 用生活场景解释这个指标衡量的是什么；
- 它的数值范围是多少？高代表什么，低代表什么？

Step 2：数学定义
- 给出完整公式，解释每个符号（包括上下标）；
- 这个指标是否受先验概率 π 影响？
  判断方法：化简公式，看分子分母是否跨越了正负样本的边界。

Step 3：指标的局限性
- 这个指标在什么情况下会失效或产生误导？
- 举一个具体的数值例子证明这种失效。

Step 4：与其他指标的关系
- 它和 <其他指标> 有什么数学关系？
  例如：微平均下 Precision = Sensitivity = F1 = Accuracy，请严格数学证明。
```

### 4.2 多分类指标体系

```
【多分类评估策略对比】

对以下每种策略，请：
A. 流程化描述计算过程（先干什么，后干什么）；
B. 举一个具体的多分类任务例子（如医学诊断、文本分类等）；
C. 数学证明其优缺点（特别是对类别不平衡的敏感性）；
D. 说明在什么场景下应该选用它。

策略列表：
1. 宏平均（Macro-average）
2. 微平均（Micro-average）
3. 多维 ROC 曲面体积（VUS）
4. <本文提出的新方法>
```

### 4.3 评估指标公理化框架

```
【主题：如何从数学上判断一个评估指标是否"好"】

1. 有哪些数学上可量化的标准可以评判一个指标的优劣？
   例如：校准性（Calibration）、一致性（Consistency）、单调性、不变性等。

2. 如果我想把现有指标的某个"零件"替换（例如把 Hellinger 距离换成其他距离），
   应该用哪些数学标准来判断新指标是否更好？

3. 在 Axiomatic Frameworks for Evaluation Metrics 这个研究领域，
   有哪些影响巨大的论文和研究组？请列举。

4. 用这套框架评价本文提出的指标：它满足哪些公理？违反了哪些？
```

## 第五层：几何与概率论补充知识

```
【主题：<填入涉及的数学概念，例如 Hellinger 距离、单纯形、球面投影、核方法>】

Step 1：几何小白版解释
- 把我当做几何小白，用最通俗的语言解释这个概念；
- 如果涉及高维几何，先从二维/三维的直觉出发，再推广到高维。

Step 2：数学定义
- 给出完整定义，解释每个符号；
- 如果有推导（例如球面方程的来历），一步一步展示，不跳过任何步骤。

Step 3：与其他概念的联系
- 这个概念和 <另一个相关概念，例如核方法、切比雪夫不等式> 有什么联系？
- 它们是否共享同一种"把低维问题投射到高维求解"的思路？

Step 4：统计距离全家桶
- 除了 <当前距离>，统计学上还有哪些距离/散度？
  请做一个综述，包括：公式、符号解释、优缺点、适用场景。
  （涵盖：L0/L1/L2 范数、Wasserstein、Jensen-Shannon、Bhattacharyya、KL 散度等）

Step 5：本文选择的理由
- 为什么本文选用了 <特定距离>？
- 它和 <另一距离> 之间有没有代数转换关系？用严格数学推导展示。
```

## 第六层：深度学习工程问题

```
【主题：GPU 加速、C++ 绑定、自定义算子】

1. 向量化与 GPU 加速
   - <具体操作> 可以在 GPU 上一次性计算吗？为什么？
   - 从 GPU 的结构和原理解释：torch 的哪些操作可以利用 GPU 加速？
   - 如何判断某个操作能否利用 GPU 加速？
   - 如何通过底层接口确认是否真正用到了 GPU？

2. 顺序依赖逻辑与 C++
   - <算法中的某步骤> 为什么无法向量化并行？
     用数学语言定义"马尔可夫/顺序依赖逻辑"，并举生活中的通俗例子。
   - 什么情况下必须编写 C++ 代码？举至少 3 个实际开发中的例子。

3. R 与 C++ 的绑定
   - R 数据通过哪个接口传入 C++ 库，计算后如何安全传回？
   - 对传回数据的格式有什么要求？
   - 在同一个 R 项目中，R 代码和 C++ 代码分别在哪些目录？
   - 给一个完整的代码例子说明绑定流程。

4. 自定义损失函数的反向传播
   - 对于本文中的自定义损失函数，反向求导是否也需要 C++ 编写？
   - 如何利用 C++ 操控 GPU 进行自定义算子的计算？给出代码示例。
```

## 附录：常用追问模板

| 触发场景 | 追问 Prompt |
|---------|------------|
| 解释太简短 | "你解释得太简短了！从头重新回答，要更详细更长，虽然你是 PhD 导师，但要像小学老师一样解释，不跳任何步骤。" |
| 公式看不懂 | "这个公式里的 <符号> 是什么？结合一个带具体数值的例子，一步一步演示如何代入计算。" |
| 概念太抽象 | "先从生活中举一个非常具体的例子，让我建立直觉，然后再给我严格的数学定义。" |
| 推导有跳步 | "这一步是怎么从上一步推出来的？不要省略，把中间所有步骤都写出来。" |
| 符号混乱 | "这里的下标 <x> 和上标 <y> 分别代表什么？在具体例子中，它们分别等于什么数值？" |
| 几乎处处 | "你说'几乎处处'，那哪些点是例外？这些例外点为什么对优化不重要？" |
| 等价性存疑 | "你说 A 和 B 等价，请从各自的定义出发，用严格数学证明，不省略任何步骤。" |
| 直觉与公式脱节 | "你给了我公式，但我还是看不出它和之前说的直觉有什么关系。能把直觉和公式一一对应地说明吗？" |
| 比较两种方法 | "A 方法和 B 方法的本质区别是什么？在什么情况下用 A，什么情况下用 B？举具体例子证明。" |
| 追问历史背景 | "这个概念/方法是谁提出的，什么时候提出的，为了解决什么问题？和它同时代的竞争方案是什么？" |

## 使用说明

1. **首次阅读**：先用"第一层：论文整体框架阅读"的 prompt，得到全貌。
2. **逐节深入**：遇到不理解的章节，按第二层 → 第五层的 prompt 逐步深入。
3. **数学推导**：遇到公式，强制要求 LLM 展示完整推导，不允许跳步。
4. **追问策略**：如果 LLM 回答太简短或跳步，直接从"附录：常用追问模板"选取追问。
5. **替换占位符**：将 `<概念名称>`、`<公式名称>`、`<算法名称>` 等替换为论文中实际内容。
6. **建立联系**：阅读后期，用"评估指标公理化框架"部分将多篇论文的方法进行横向比较。
